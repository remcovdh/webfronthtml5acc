What is HTML5 (day 2, part 1)

HTML5 is browser driven change on the HTML specification. In reality this means features are developed by browser vendors (and community developers) which end up to become standardized and formalized in a later stage (or not). Therefore there was no clear specification on what is part of HTML5 on what is not for a long time. There was however a growing concessus of things that are clearly seen as part of it and what is not. Next to that there is a continuous different opinion between vendor browsers resulting in different implementations.

A long list of the differences can be found at http://caniuse.com/. A recent blog post about the status of the html5 standardization proces state can be found at http://www.w3.org/blog/2014/06/html5-on-our-way-to-recommendation/

The W3C organization categorized the different changes that are (or beter can become) part of the HTML5 spec are:
  • semantics
  • css3
  • multimedia
  • graphics and 3D
  • device access
  • performance
  • offline storage
  • connectivity

As the title of this second part of this course already suggested HTML is not one specification or API or library. It is a wild west. In a nice graphical overview on the wikipedia helps in getting some overall view on this, although far not as up-to-date as the http://caniuse.com/ site.
