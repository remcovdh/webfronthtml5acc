Equality operators: About using the strict equality version (triple equal sign): Zakas says: "This helps to maintain data type integrity throughout your code.". See more information on the two type of equality https://developer.mozilla.org/en/docs/Web/JavaScript/Guide/Sameness. This is so because in the strict equality operator both the value and the type must be the same.

In JavaScript, the following values are considered to be falsy values:

  • false
  • 0
  • null
  • "" (empty string)
  • NaN (Not a Number)
  • #ff0000
All other values of JavaScript can be safely assumed to be truthy values. Even an empty function, empty array and empty objects are also truthy.

Ecmascript

Deadly naming. Was only a working title but did not change later. Javascript the name which most people used was owned by Sun.

Ecma is the standardisation institution, because at the time the World Wide Web Consortium (W3C) did not want to set this standard.

Current version used by most browser is ECMA-262, edition 5. And in Chrome and Firefox strive towards version 6.

The Sixth Edition adds significant new syntax for writing complex applications, including classes and modules, but defines them semantically in the same terms as ECMAScript 5 strict mode. Other new features include iterators and for/of loops, Python-style generators and generator expressions, arrow functions, binary data, collections (maps, sets and weak maps), and proxies (metaprogramming for virtual objects and wrappers).
