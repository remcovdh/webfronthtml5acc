Mobile (day 3, part 4)

Mobile first

There are a view different opinion on how the design would work for a website. Should it design for mobile first? When should you decide to make different versions of the website?

Takeway from https://www.google.com/events/io/io14videos/95a7ccd2-38bf-e311-b297-00155d5066d7 See also http://bit.ly/mobilewebprinciples

Home page and navigation:

  1. Keep call to action front and center
  2. Logo is home
  3. Menu short and sweet no scrolling should be necessary

Search should be visible:

  1. Smart search (completing, spelling correction result, context aware search (not just google))
  2. Never zero in the result list (should be clear while searching)
  3. Select instead of navigate (in case this can be done in view select questions)

Let user:

  1. Purchase as guest
  2. Don't ask personal information up front
  3. Picking up where left off (consistent url policy)

Design efficient form (it is already painfull enough):

  1. Auto filling form
  2. Use pre existing
  3. Minimize tabs and handling
  4. Real-time validation

Side wide considerations:

  1. Don't make users pinch and zoom
  2. Provide expandable / tabble images
  3. Be clear in why you need your users location
  4. Keep browser in your single browser window
  5. Don't let promotions steal the show
  6. Optimize your entire site for mobile
