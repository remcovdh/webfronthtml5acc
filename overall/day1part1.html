JSLint and JSHint

One of the godfathers in the javascript community and javascript architect at Yahoo, Douglas Crowford, ones wrote jslint. That was a configurable parser/code checker. It contained a very opinionated view on which javascript contructs should and should not be used. To see it work http://www.jslint.com/.

JavaScript parser that can analyze a JavaScript program and report on the bad parts that it contains. JSLint pro- vides a degree of rigor that is generally lacking in JavaScript development. It can give you confidence that your programs contain only the good parts.

Douglas Crowford is also writer of the rather small book: "javascript the good parts".

Version control

The main tool for doing version control of this moment, especially in the opensource world is git. Git is a so called distributed version control system with strong emphasis on branching and merging instead of locking. A very popular server version where sources can be easily shared is github. We will use this for this course. See github for more information. A flexible versioning is very important for a modern workflow and testdriven development. It helps you try things and makes merging new and refactored code into your central reporository easier.

All accumulated information on this course is also collected on github.

The basic set of commands for git during this course:

  • git clone
  • git add
  • git commit
  • git pull
  • ..

The grunt.js code 
// Where did we get the information
// http://gruntjs.com/getting-started grunt in general
// https://github.com/gruntjs/grunt-contrib-watch for watching and to combine with reload
// https://github.com/gruntjs/grunt-contrib-livereload reload hooks

module.exports = function(grunt) {

	// Project configuration.
	grunt.loadNpmTasks('grunt-contrib-connect');
	grunt.loadNpmTasks('grunt-contrib-watch');	
	
	grunt.initConfig({
		connect: {
			server: {
				options: {
					// keepalive:true, grunt:connect:keepalive of zoiets kan nog wel
					livereload: 35729,
					port: 9002
				}
			},
		},
		watch: {
			livereload: {
				options: {
					livereload: 35729
				},
				files: [
					'{,*/views/}*.html',
					'app/{,*/}*.js'
				]
			}
		}
	});

	grunt.registerTask('default', 'Starting server and checking file changes', function() {
		grunt.log.write('Starting server\n');
		grunt.task.run('connect');
		grunt.log.write('Server started\n');
		grunt.log.write('Watch html and js file changes\n');
		grunt.log.write('-html only root, /views/ and one subdir down\n');
		grunt.log.write('-js files only in app and one subdir down\n');
		grunt.task.run('watch');
	});

	grunt.registerTask('another', 'Log some stuff.', function() {
		grunt.log.write('Logging some stuff...').ok();
	});

};
